# Vibe Runner — Enforcement Meta-Rule

## Принцип

Ты знаешь правила пользователя (из rules/, CLAUDE.md, agents/).
Твоя задача — следовать им **СТРОГО**, без исключений.

Этот meta-rule усиливает ВСЕ остальные правила. Он не добавляет новых требований — он ЗАСТАВЛЯЕТ соблюдать существующие.

## Onboarding (первый запуск)

При старте сессии hook автоматически запускает discovery и создаёт файл `vibe-runner-discovery.json`.

Если этот файл существует:
1. **Прочитай** `vibe-runner-discovery.json`
2. **Прочитай СОДЕРЖИМОЕ** каждого найденного инструмента:
   - Каждый rule из `rules_global` и `rules_project`
   - `CLAUDE.md` (global и project)
   - Каждый субагент из `subagents_global` и `subagents_project`
3. **Сформируй понимание** процесса пользователя:
   - Какие ПРАВИЛА описаны (TDD, conventional commits, security...)
   - Какие ФАЗЫ описаны (если есть)
   - Какие ПРОВЕРКИ автоматические (hooks) vs ручные
   - Какие MCP/LSP дают дополнительные возможности
4. **Покажи** пользователю: «Вот что я понял о твоём процессе: ...»
5. **Спроси** подтверждение
6. После подтверждения — **активируй enforcement**

## Самокоррекция

ЕСЛИ ты нарушаешь ЛЮБОЕ правило из rules/ или CLAUDE.md:

1. **ОСТАНОВИСЬ** немедленно
2. Сообщи: `[VR:INCIDENT] {какое правило нарушено} | {что произошло}`
3. **Вернись** к правильному порядку действий
4. Продолжи работу по процессу
5. Выведи compliance маркер

**НИКОГДА** не продолжай работу при обнаруженном нарушении — сначала исправь.

## Compliance маркеры

После КАЖДОГО значимого действия (запись файла, коммит, завершение этапа) выведи:

```
[VR] {контекст} | {правило_1} {статус} | {правило_2} {статус}
```

Статусы:
- ✅ — соблюдено
- ❌ — нарушено → исправлено
- ⏭️ — не применимо к данному действию

Примеры:
```
[VR] Write src/api.py | TDD ✅ | Anti-Mirage ✅ | Security ✅
[VR] git commit | Conventional ✅ | Self-review ✅
[VR:INCIDENT] TDD ❌ → написал код без теста → откатил → написал тест → продолжил
```

## Compliance log

Каждый маркер дублируй в файл `vibe-runner.log`:

```
[2026-03-10 14:30:15] [VR] Write src/api.py | TDD ✅ | Anti-Mirage ✅
[2026-03-10 14:31:02] [VR:INCIDENT] TDD ❌ → исправлено
```

## Что НЕ делает этот meta-rule

- НЕ добавляет новых правил — только усиливает существующие
- НЕ модифицирует файлы пользователя — только читает
- НЕ блокирует действия — только корректирует и логирует
- НЕ отправляет данные наружу — всё локально
